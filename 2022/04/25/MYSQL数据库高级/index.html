一、数据查询🍿

1.联合查询

UNION

合并两个或多个SELECT语句的结果集 (●ˇ∀ˇ●)就是同时把两个表里面同一个字段同时显示

2.不重复查询

DISTINCT

重复值将不显示

LIKE

“%”和”_”

4.排序查询

ORDER BY

默认为升序，降序后加desc。

升序 (ASC)或降序 (DESC)

5.限制查询

LIMIT

查几条，或者从第几条到第几条。

6.聚合

SUM(求和)、COUNT(*)(记录数)、MAX(最大值)、MIN(最小值)

7.连接
  • 内连接

内连接查询指所有查询的结果都能够在连接的表中有对应记录。

  • 左连接和右链接

LEFT JOIN RIGHT JOIN

左（右）连接指以左（右）边为基准，来匹配右（左）边表的数据。（未匹配为NULL）

二、事务🌭

1.事务概述

暂存点做一个处理。

事务为一组 SQL 语句逻辑，SQL 语句要么全执行，要么全不执行

2.事务特性

ACID

  • 原子性(Atomicity)

  • 一致性(Consistency)

  • 隔离性(Isolation)

  • 持久性(Durability)

3.事务提交

执行操作和提交执行结果

默认为自动提交

4.事务回滚

系统故障，部分执行不成功，执行成功的退回到未执行。

三、数据库索引🥪

1.索引概述

目录写页码

索引类型 特点
普通索引 任何字段都可以创建
唯一索引 字段的值必须具有唯一性才可以创建，效率高于普通索引
主键索引 主键的值必须具有唯一性才可以创建
全文索引 只针对文本类型
复合索引 多个字段联合创建
2.BTree索引和Hash索引

  • BTree 索引

    **二叉树查找 **

    能加快数据的访问速度，因为存储引擎不再需要进行全表扫描来获取数据，数据分布在各个节点之中。

    如果要查找中文需要用到Sphinx（斯芬克斯）/Coreseek 技术来处理中文

  • Hash 索引

    无法满足范围查找。

    只有精确匹配索引所有列的查询才有效

4.唯一索引和联合索引

UNIQUE

创建唯一索引不仅可以提高访问速度，还可以避免数据出现重复。唯一索引可以有多个，但索引列的值必须唯一，同时索引列的值允许有空值。

联合查询又叫复合索引，将两个或更多个列的索引称作复合索引。

一个查询可以支持”a”,”a,b”,”a,b,c””3种组合进行查找。从左往右使用索引中的字段，一个査询可以只使用索引中的一部分，但只能是最左侧部分。

5.EXPLAIN 语句

EXPLAIN

模拟优化器执行 SQL 语句,以显示 MySQL 如何使用索引来处理 SELECT 语句和连接表，可以
帮助我们选择更好的索引和写出更优化的査询语句。

7.MySQL优化

  • 索引优化

    1. 只要列中含有 NULL 值，就最好不要在此列设置索引，这是因为在复合索引中如果有 NULL 值，则此列在使用时不会使用索引。
    2. 尽量使用短索引,可以设置前缀长度。
    3. 最佳左前缀法则，若多列，查询要从最左前列开始且不跳过索引中的列。
    4. 尽量不在索引列上做任何运算操作，会导致索引失效而转向全表扫描。
    5. 储存引擎不能使用索引中范围条件右边的列。
    6. 尽量使用覆盖索引，减少“select*”。
    7. “IS NULL”“IS NOT NULL” 也无法使用索引。
    8. 对于 LIKE 语句，以通配符开头会使索引失效并变成全表扫描的操作，以”％”结尾会使用索引。
    9. 字符串不加单引号，索引会失效。
    10. 尽量不要使用“not in”和“<>”操作。

  • SQL语句优化

    1. 在查询时，能不用“”就不用“”，尽量写全字段名。
    2. 在多表连接时，尽量利用小表驱动大表，即小表 JOIN 大表。
    3. 在千万级分页时，尽量使用 LIMIT 语句。

  • SQL表优化

    1. 表的字段尽可能用 NOT NULL。
    2. 字段长度固定的表在査询时会更快。
    3. 把数据库的大表按时间或一些标志分成小表。
    4. 将表分区。
