MySQL数据库高级👕

[^]:

一、数据查询🍿

1.联合查询

UNION

合并两个或多个SELECT语句的结果集 (●ˇ∀ˇ●)就是同时把两个表里面同一个字段同时显示

2.不重复查询

DISTINCT

重复值将不显示

3.模糊查询

LIKE

“%”和”_”

4.排序查询

ORDER BY

默认为升序，降序后加desc。

升序 (ASC)或降序 (DESC)

5.限制查询

LIMIT

查几条，或者从第几条到第几条。

6.聚合

SUM(求和)、COUNT(*)(记录数)、MAX(最大值)、MIN(最小值)

7.连接
  • 内连接

内连接查询指所有查询的结果都能够在连接的表中有对应记录。

  • 左连接和右链接

LEFT JOIN RIGHT JOIN

左（右）连接指以左（右）边为基准，来匹配右（左）边表的数据。（未匹配为NULL）

二、事务🌭

1.事务概述

暂存点做一个处理。

事务为一组 SQL 语句逻辑，SQL 语句要么全执行，要么全不执行

2.事务特性

ACID

  • 原子性(Atomicity)

  • 一致性(Consistency)

  • 隔离性(Isolation)

  • 持久性(Durability)

3.事务提交

执行操作和提交执行结果

默认为自动提交

4.事务回滚

系统故障，部分执行不成功，执行成功的退回到未执行。

三、数据库索引🥪

1.索引概述

目录写页码

索引类型 特点
普通索引 任何字段都可以创建
唯一索引 字段的值必须具有唯一性才可以创建，效率高于普通索引
主键索引 主键的值必须具有唯一性才可以创建
全文索引 只针对文本类型
复合索引 多个字段联合创建
2.BTree索引和Hash索引

  • BTree 索引

    **二叉树查找 **

    能加快数据的访问速度，因为存储引擎不再需要进行全表扫描来获取数据，数据分布在各个节点之中。

    如果要查找中文需要用到Sphinx（斯芬克斯）/Coreseek 技术来处理中文

  • Hash 索引

    无法满足范围查找。

    只有精确匹配索引所有列的查询才有效

3.索引的建立和使用

CREATE TABLE

可以在创建表的同时进行创建，也可以在创建表之后进行添加。

也可以使用 ALTER TABLE 语句在创建表之后添加索引。还可以使用 CREATE INDEX 语句添加索引。
