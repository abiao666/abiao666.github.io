开启调试与配置文件

[^]:

一、开启调试

  1. 调试模式开启后，会牺牲一些执行效率，但大大提高了开发排错的能力
  2. 当项目部署到生产环境时，再关闭调试模式即可
  3. 安装好的 TP6.0 默认没有开启调试
  4. 根目录.example..env 文件是环境配置文件，删除前面的 .example. 即可生效
  5. 右下角会出现 Trace 调试小图标，说明调试开启了
  6. 查看.env文件，打开调试的环境变量为 APP_DEBUG = true
  7. 开启调试模式的优势
    1. 会记录系统运行流程的执行过程
    2. 展示错误和调试信息，并开启日志记录
    3. 模板修改可以及时生效（不会被缓存干扰）
    4. 启动右下角的 Trace 调试功能，更加强大
    5. 发生异常时，也会显示异常信息

二、配置信息

  1. 配置文件有两种形式，开启调试采用的 .env 文件时一种，适合本地
  2. 另一种配置文件，在根目录下的 config 里，有很多类型的配置，适合部署
  3. .env 环境变量用于本地开发测试，部署后会被忽略
  4. 这两个配置文件的优先级，在本地测试时，.evn 优先于 config
  5. 从 config 配置中可以看出，它是优先读取 .env 的，然后再默认配置一个自己的
  6. 而到了部署环境，.env 会被忽略，则自动切换到 congig配置
