Larevel 目录结构

[^]:

1、目录结构

app💨

应用程序核心目录，几乎项目所有的类都在这里。写我们的模型，控制器。

bootstrap

包含框架启动文件 ‘app.php’ ，和启动时为了优化性能而生成的文件。

  • cache

用来存放启动时优化性能时而生成的缓存文件

[^必须设置 web 服务器可读写]:

config

包含所有配置文件。

database

包含数据库填充、迁移、模型工厂文件，可以用作 ‘SQLite’ 数据库存放目录。

public

静态资源目录，并包含了首页文件 ‘index.php’。（根目录）

resources💨

包含了为编译的源文件（模板、语言、资源），写我们的模板。

routes💨

包含了所有的路由定义，写我们的路由。

storage

包含了编译好的模板文件，session文件、缓存文件、日志等文件。

用户上传的文件、报错日志

[^必须设置 web 服务器可读写]:

tests

单元目录，包含了自动测试文件。

vendor

composer 依赖目录，第三方安装包都会存放在这里。
